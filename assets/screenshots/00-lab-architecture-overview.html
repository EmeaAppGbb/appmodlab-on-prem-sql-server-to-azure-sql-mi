Lakeview Medical Center — SQL Server → Azure SQL MI Migration Lab

🏥 Lakeview Medical Center

On-premises SQL Server 2016 → Azure SQL Managed Instance migration lab

🗄️4 Databases

  • PatientDB — 16 tables, 8 views, 8 sprocs, 9 functions
  • BillingDB — 9 tables, 5 cross-DB stored procs
  • ReportingDB — 5 cross-DB aggregate views
  • SchedulingDB — 5 tables, room & appointment mgmt

⚠️Migration Challenges

  • Cross-database queries (PatientDB ↔ BillingDB)
  • CLR assemblies (.NET medical calculations)
  • Service Broker async messaging
  • Linked servers (4 external systems)

✅Supported on MI

  • SQL Agent jobs (7 jobs + 4 alerts)
  • Service Broker (within instance)
  • CLR (with TRUSTWORTHY or signing)
  • TDE encryption (built-in on MI)

🚫Not Supported on MI

  • Linked servers to on-prem systems
  • Cross-database queries (3-part names)
  • SQLNCLI11 / OraOLEDB providers
  • UNC backup paths (\\BACKUP-SERVER\)
